# bilateral
В данной лабораторной работе был реализован алгоритм билатеральной фильтрарции

На CPU для каждого из N×M пикселей вычисляется окно 3×3, итого N×M×9 операций выполняются последовательно.
На GPU каждый пиксель изображения обрабатывается отдельным потоком одновременно. Изображение делится на блоки 16×16 потоков, и все блоки выполняются параллельно.

В ходе лабораторной работы было замерено время выполнения функций. Использование распараллеливание потока дает выигрыш в 150 раз. 0.002 секунд на GPU против 0.3 на CPU. Для чистоты эксперимента использовалась библиотека Numba - библиотека для Python, которая компилирует функции в машинный код прямо во время выполнения. Декоратор @njit при первом вызове компилирует функцию в нативный код. При последующих вызовах уже выполняется скомпилированная версия.
